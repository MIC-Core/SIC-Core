== Subset of SIC-Core for basic usecases
    
The information and metadata required depend on the use case. The following proposes a set of metadata that enables basic functionality for search, analysis, reuse, and traceability. A key factor for user acceptance is that most of this information can be provided by the used IT systems. 
The Administrative Data category contains information used to identify and organize the simulation. 

This set of metadata can be used as starting point in implementation metadata for simulation.

[cols="1>m,2^e,7<",width="100%",align="center",frame="topbot",options="header"]
|===
| Attribute Name| Category | Obligation | Description | Typical Source
| Ordering project | Administrative |  Recommended | Unique name of the ordering project | Project management tool
| Specific Product | Administrative |  Recommended | Unique name of the specific product | Project management tool
| Simulation Request | Administrative |  Recommended | Unique name of the Simulation Request | Project management tool
| Simulation task | Administrative | Mandatory | Unique name of the simulation task | Project management tool
| Item under Test | Analysis | Mandatory | System, component, or product being analyzed | Tool / system
| Simulation Objective | Analysis |  Recommended | Objective or purpose of the simulation | User
| Quality criterion | Mandatory | Recommended | Criteria used to evaluate simulation success | User
| Used Model | Execution | Mandatory | Model used for the simulation | Tool / system
| Used Parameter | Execution | Mandatory | Parameter used for the simulation | user (selection)
| Used Tool | Execution | Mandatory | Tool used for the simulation | Tool / system
| Task status | Fulfillment | Mandatory | Is the task still in progress abortet, finished | User
| Credibility fulfilled | Fulfillment | Recommended | Documentation of the credibility fulfillment decision | User
| Objectives fulfilled | Fulfillment |  Recommended | Documentation of the objectives fulfillment decision  | User
|===
